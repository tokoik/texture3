# texture3 - 第７回 テクスチャ座標の自動生成 サンプルプログラム

## 1. 概要

このプログラムは、OpenGL における「テクスチャマッピング (Texture Mapping)」の基礎を学ぶための、学生向けのサンプルプログラムです。本プログラムは、以下のブログ記事の解説に沿って学習を進めるための雛形として提供されています。

- [第７回 テクスチャ座標の自動生成](https://tokoik.github.io/blog/%E3%83%86%E3%82%AF%E3%82%B9%E3%83%81%E3%83%A3%E5%85%A5%E9%96%80/2004/12/28/texture.html)

今回はテクスチャマッピングにおいて、**テクスチャ座標を自動生成する仕組み**について学びます。

通常、物体の表面にテクスチャ（画像）を貼り付けるには、各頂点に対応するテクスチャ上の座標（s, t）をプログラム内で手動で割り当てる必要があります。しかし、OpenGL の「テクスチャ座標の自動生成機能」を使うことで、頂点の3次元座標（オブジェクト空間または視野空間の座標）から直接テクスチャ座標を自動的に計算させることができます。

プログラムを実行すると、表面にタイヤのパターン画像（tire.raw）が遠近感を持って投影された箱が表示され、テクスチャ自体が自動で回転するアニメーションが描画されます。また、マウス操作で箱全体を回転させることで、投影されたテクスチャがオブジェクトの動きに追従する様子を確認できます。

![テクスチャ座標の自動生成](https://tokoik.github.io/blog/assets/images/texture3.gif)

## 2. ビルド方法

このプログラムは CMake を用いてビルドを構成しています。各プラットフォームごとの手順は以下の通りです。なお、プログラムをビルドするためのバイナリディレクトリは、バージョン管理ファイル（.gitignore）の設定に合わせて build という名前にします。

### 2.1 Windows (Visual Studio 2022 の場合)

1. コマンドプロンプトまたは PowerShell を開き、このプロジェクトのディレクトリに移動します。

2. 以下のコマンドを実行してビルドディレクトリを作成し、CMake で構成を行います。

```bat
mkdir build
cd build
cmake .. -G "Visual Studio 17 2022"
```

3. 生成された build フォルダ内の texture3.sln を Visual Studio で開きます。

4. ソリューションエクスプローラーで texture3 プロジェクトを右クリックし、「スタートアップ プロジェクトに設定」を選択します。

5. 「ローカル Windows デバッガー」をクリックするか、F5 キーを押してビルドおよび実行します。

### 2.2 macOS (Xcode の場合)

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。

2. 以下のコマンドを実行してビルドディレクトリを作成し、Xcode 用のプロジェクトを生成します。

```sh
mkdir build
cd build
cmake .. -G Xcode
```

3. 生成された build/texture3.xcodeproj を Xcode で開きます。

4. 左上のスキーム選択（再生ボタンの横）が texture3 になっていることを確認します。

5. 「Run」ボタン（再生ボタン）をクリックするか、Command + R を押してビルドおよび実行します。

### 2.3 Ubuntu Linux

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。

2. 必要なパッケージ（freeglut3-dev や pkg-config など）がインストールされていることを確認し、以下のコマンドでビルドします。

```sh
mkdir build
cd build
cmake ..
make
```

## 3. 使い方

### 3.1 プログラムの起動方法

各OSとも、ビルド後に生成されるバイナリディレクトリ (build) やそのサブフォルダから起動します。（※ CMake の設定により、Windows や Xcode では Debug などのフォルダ下に実行ファイルが置かれることがあります）

- **Windows**

Visual Studio 上で「ローカル Windows デバッガー」をクリックして実行するか、またはコマンドプロンプトから以下のコマンドで起動します。

```cmd
cd build\Debug
texture3.exe
```

- **macOS**

Xcode 上で左上の「Run（再生ボタン）」をクリックするのが楽です。これにより texture3.app アプリケーションバンドルとして自動的に実行されます。アプリケーションバンドルを直接起動するなら、Finder から build/Debug/texture3.app をダブルクリックするか、ターミナルから open build/Debug/texture3.app を実行します (この場合はエラーメッセージ等が表示されません)。

- **Ubuntu Linux**

ターミナルから以下のコマンドで実行ファイル（バイナリ）を直接起動します。

```sh
cd build
./texture3
```

### 3.2 操作方法

- **マウスの左ボタンでドラッグ**: 画面内のオブジェクト（箱）を３次元的に回転させることができます。

- **キーボードの q, Q または ESC キー**: プログラムを終了します。

## 4. 解説

このプログラムは、テクスチャ座標を明示的に指定せずにテクスチャをマッピングします。

### 4.1 頂点座標とは独立したテクスチャ処理 (box.cpp)

- 以前のプログラムでは、箱を描画する際に `glTexCoord2dv()` 等を用いて頂点ごとのテクスチャ座標を記述していました。

- このプログラムの `box()` 関数では、その記述を削除（または `#if 0` ～ `#endif` で無効化）しており、**純粋な頂点座標と法線ベクトルのみ** を使って描画しています。

### 4.2 自動生成パラメータの定義と初期設定 (`init()` 関数)

- **`init()`** 関数内では、手動設定の代わりにテクスチャ座標の自動算出を設定します。

- **`glTexGeni(` `GL_S`, `GL_TEXTURE_GEN_MODE`, `GL_OBJECT_LINEAR` `)`** によって、頂点のオブジェクト空間（ローカル座標系）での座標値を用いて、テクスチャ座標を生成するよう指示します。

- **`glTexGendv(` `GL_S`, `GL_OBJECT_PLANE`, `genfunc[0]` `)`** によって、座標を算出するためのパラメータ（係数）を与えます。頂点の座標を `(x, y, z, w)`、パラメータを `params` とすると、テクスチャ座標 `g` は内積計算 `g = x*params[0] + y*params[1] + z*params[2] + w*params[3]` で求まります。

- `r` と `q` (3次元と斉次座標のテクスチャ要素) についても同様にパラメータを設定しておくことで、後のテクスチャ変換行列を適用した際に破綻しないようにしています。

### 4.3 自動生成機能の有効化 (`scene()` 関数)

- 箱を描画する直前に、`glEnable(` `GL_TEXTURE_GEN_S` `)` などを用いて座標自動生成機能を有効にします。これによって、後続の `box()` 描画時に自動計算がはたらき、テクスチャが貼り付けられます。

### 4.4 テクスチャの投影とアニメーション (`display()` 関数)

自動生成されたテクスチャ座標に対して、さらに行列計算で操作を加えることができます。

- **モデルの配置と回転**: `glMatrixMode(` `GL_MODELVIEW` `)` では、`gluLookAt` の代わりに `glTranslated` を用いて視点を奥へ移動し、`trackballRotation()` を掛けることで、マウス操作により「箱自体」を回転できるようにしています。

- **テクスチャ行列による投影とアニメーション**: `glMatrixMode(` `GL_TEXTURE` `)` に切り替えて変換を行うと、テクスチャ（画像）側だけを移動・回転・伸縮させることができます。本プログラムでは `glRotated` による時間経過の回転アニメーションに加え、`gluPerspective` や `gluLookAt` を用いて投影変換や視点変換を行列として組み込むことで、テクスチャが遠近感を持った状態で投影される「投影マッピング」を実現しています。

- **オブジェクト座標系での自動生成 (`GL_OBJECT_LINEAR`) の効果**: `init()` でテクスチャ座標の生成基準に `GL_OBJECT_LINEAR` (オブジェクト平面) を指定しているため、テクスチャ座標はモデルビュー変換の「前」のローカル座標から算出されます。このため、マウスで箱を回転させると、投影されたテクスチャも箱に追従して一緒に回転します。
