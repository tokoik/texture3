# テクスチャマッピング サンプルプログラム（テクスチャ座標の自動生成）

このプログラムは、ブログ記事「[テクスチャ第７回：テクスチャ座標の自動生成](https://tokoik.github.io/blog/opengl/%E3%83%86%E3%82%AF%E3%82%B9%E3%83%81%E3%83%A3/2004/12/28/texture.html)」の解説用サンプルコードです。

## 1. 概要

本プログラムは、3Dコンピュータグラフィックスにおける「テクスチャマッピング」において、**テクスチャ座標を自動生成する仕組み** を学ぶためのものです。

通常、物体の表面にテクスチャ（画像）を貼り付けるには、各頂点に対応するテクスチャ上の座標（`s, t`）をプログラム内で手動で割り当てる必要があります。しかし、OpenGL の「テクスチャ座標の自動生成機能」を使うことで、頂点の3次元座標（オブジェクト空間または視野空間の座標）から直接テクスチャ座標を自動的に計算させることができます。

プログラムを実行すると、表面にタイヤのパターン画像（tire.raw）が貼り付けられた箱が表示され、そのテクスチャが自動でアニメーション（回転と奥行きの変化）します。テクスチャの投影や切り出しを数式で表現する仕組みを視覚的に確認できます。

## 2. ビルド方法

CMake を使って各 OS 向けのビルド設定を生成します。ソースコードのあるディレクトリでターミナルを開き、以下を実行して build ディレクトリ内でビルドを行ってください。

### Windows (Visual Studio 2022) の場合
```cmd
> mkdir build
> cd build
> cmake .. -G "Visual Studio 17 2022"
```
生成された build フォルダ内の texture3.sln を Visual Studio で開きます。「ソリューションエクスプローラー」から texture3 を右クリックして「スタートアップ プロジェクトに設定」し、ローカル Windows デバッガーで実行してください。

### macOS (Xcode) の場合
```bash
$ mkdir build
$ cd build
$ cmake .. -G Xcode
```
生成された build/texture3.xcodeproj を Xcode で開き、左上の「Run」ボタン（再生ボタン）から実行してください。

### Ubuntu Linux の場合
※事前に freeglut3-dev と pkg-config をインストールしてください。
```bash
$ mkdir build
$ cd build
$ cmake ..
$ make
```

## 3. 使い方

### プログラムの起動方法
- **Windows:** Visual Studio から実行、または build\Debug\texture3.exe を起動。
- **macOS:** Xcode から実行、または build/Debug/texture3.app を起動。
- **Ubuntu Linux:** ターミナルから ./texture3 で起動。

### 操作方法
- **視点の回転 (トラックボール操作):** 画面を **マウスの左ドラッグ** すると、箱を任意の方向に回転させて様々な角度から観察できます。
- **終了:** キーボードの q、Q、または ESC でプログラムを終了します。

## 4. 解説

ブログ記事の内容に基づいた、このプログラム内の重要な処理のアルゴリズムは以下の通りです。

### 頂点座標とは独立したテクスチャ処理 (box.cpp)
以前のプログラムでは、箱を描画する際に `glTexCoord2dv()` 等を用いて頂点ごとのテクスチャ座標を記述していました。本プログラムの `box()` ではその記述を削除（または `#if 0` で無効化）しており、**純粋な頂点座標と法線ベクトルのみ** を使って描画しています。

### 自動生成パラメータの定義と初期設定 (`init()` 関数)
`init()` 内では、手動設定の代わりにテクスチャ座標の自動算出を設定します。

- **`glTexGeni(GL_S, GL_TEXTURE_GEN_MODE, GL_OBJECT_LINEAR)`** 
  頂点の「オブジェクト空間（ローカル座標系）」での座標値を用いて、テクスチャ座標を生成するよう指示します。（視点系の `GL_EYE_LINEAR` もあります）
- **`glTexGendv(GL_S, GL_OBJECT_PLANE, genfunc[0])`** 
  座標を算出するためのパラメータ（係数）を与えます。頂点の座標を `(x, y, z, w)`、パラメータを `params` とすると、テクスチャ座標 `g` は内積計算「`g = x*params[0] + y*params[1] + z*params[2] + w*params[3]`」で求まります。
- `r` と `q` （3次元と斉次座標のテクスチャ要素）についても同様にパラメータを設定しておくことで、後のテクスチャ変換行列を適用した際に破綻しないようにしています。

### 自動生成機能の有効化 (`scene()` 関数)
箱を描画する直前に、`glEnable(GL_TEXTURE_GEN_S)` などを用いて座標自動生成機能を有効にします。これによって、後続の `box()` 描画時に自動計算がはたらき、テクスチャが貼り付けられます。

### テクスチャの投影とアニメーション (`display()` 関数)
自動生成されたテクスチャ座標に対して、さらに行列計算で操作を加えることができます。
- `glMatrixMode(GL_TEXTURE)` に切り替えて変換を行うと、テクスチャ（画像）側だけを移動・回転・伸縮させることができます。
- `gluPerspective` や `gluLookAt` を用いて投影変換や視点変換を行列として組み込むことで、テクスチャが遠近感を持った状態で投影される「投影マッピング」の動作アニメーションが実現されています。
